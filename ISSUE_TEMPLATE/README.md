# Issue Templates

This directory contains templates for creating standardized issues in this repository.

## Available Templates

### Bug Report (`bug_report.yml`)
Use this template to report bugs or unexpected behavior. The form includes:
- Contact information
- Bug description
- Version information
- Browser compatibility
- Log output
- Code of Conduct agreement

### Feature Request (`feature_request.yml`)
Use this template to suggest new features. The form includes:
- Problem description
- Proposed solution
- Alternative solutions
- Priority level
- Additional context

## Configuration

The `config.yml` file controls:
- Whether blank issues are allowed
- Contact links for questions and documentation

## How to Use

1. Go to the Issues tab
2. Click "New Issue"
3. Select the appropriate template
4. Fill out the required fields
5. Submit your issue

## Customization

You can customize these templates by:
1. Editing the `.yml` files
2. Adding new templates
3. Modifying the `config.yml` settings

For more information, see [GitHub's documentation on issue templates](https://docs.github.com/en/communities/using-templates-to-encourage-useful-issues-and-pull-requests).
